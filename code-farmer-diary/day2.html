# 回顾生涯

回顾回顾职业生涯学过的技术顺便做个记录。
忘记谁说的了“教别人技术能够更快的掌握技术”，我觉得确实是有道理的，我决定按照教我孩子的思路写文档，说不定哪天他们真的会用到呢😂。
从我学习的经历来看，市面上教材很多，但是案例却很少，最起码我自己学习的时候不善于找合适的案例，现在准备多写一点案例，我觉得这才是进阶的法门。
最初入行的时候一直期盼着进了公司可以找一个内行的同事做师傅可以带带我，但是一路走来我发现大家知道的都差不多，甚至还不如我所学的知识，而且我还觉得他们说的还是错误的，我也有这毛病，就是希望找到最完善的文档最官方的来源。
我在学习的时候用过很多的网站，从最初开始学html、javascript的时候使用的w3schoolopen in new window这个网站使用asp制作的，年代久远了，但是当时我就只知道这一个网站，里面的内容简洁有样例，就这样我开始了我的编程学习。
后来手机上学习比较多找到了一个类似的网站编程狮open in new window当时两个网站很像，不知道从哪一天开始编程狮的风格变了，对手机端更友好，于是我就开始用编程狮了。再后来我顺着藤摸到了瓜，我好像发现菜鸟教程open in new window是相同的作者。菜鸟教程更简洁，后来我手机端就只看菜鸟了。
这几个网站都在改进，w3school也在悄悄的发生变化，虽然依然对移动端不够友好，但是内容确实偷偷的增多了。编程狮更加商业化了，菜鸟教程也有在不断推出新的教程，入门确实很棒，但是后来我发现这些只是搬运工，让我有一个集中学习的地方。再后来我又去了廖雪峰open in new window阮一峰open in new window大神们的网站学习，受益匪浅，很多知识都学习了很多遍。
从html到css我看过好多版本的，后来有了html5，我又重新看了一遍。通过看《禅意花园》和一些其他的布局类的书籍我初步的完成了一些简单布局。很多知识都是学过之后用到了，再到runoob菜鸟教程上进行参考进行巩固的。 javascript的学习我就学的次数就更多了，从最初的刚从学校毕业的时候只会c++的一些皮毛，了解了编程的基础语法，再到学校里主修的vb还有java，很多语法都很相似。看多了练习的少了反而都记忆出现了混淆。
javascript先是看的w3school，再看的菜鸟教程，后来工作中看了jquery，以及vuejs（当时我并不知道官网的文档更好），再到出现了es5语法后MDN Web Docopen in new window开始完善中文文档，我觉得官网的肯定都是有精髓的，于是又看了一遍mdn上的知识，自此查阅文档电脑端主要就看mdn了，到现在为止文档一直在完善，增加了很多的功能，不知道为什么看着这些文档就觉得挺兴奋的，感觉有看头，特别是从业相关技术的时候，会查看这个技术兼容性如何普及度如何，我在公司是否能够使用等等。
