# 搭建vuepress博客

# 起因

刚开始觉得vuepress的这个好简陋，现在看看还挺耐看的。

需要隐藏贡献人，毕竟就我自己一个人在写博客。

我发现还是github上的pages功能好用直接重新推送就能够实现重新部署，但是在gitee上我尝试了下好像无法达到目的，只能够点击界面上的更新按钮才能更新，不知道是不是我的使用方式有问题。
我要放弃了，经过百度发现gitee上的自动更新功能是付费功能，当然还有其他非正统方式，但是官方都不免费支持这种方式就有点不友好了，我还是主要发布到github上吧，哪天我想起更新了gitee就更新下好了。

# 渐进式更新

现在将日记写到git的私有仓库内，然后将日记生成的html写到github.io上，目前感觉听方便的了。
现在只需要npm run deploy即可更新博客，还能保存现有的进度，缺点就是没有留言功能，问题不大，写给自己看的。 现在已经将完整的框架搭建起来了，只需要慢慢的更新日常就行了，挺棒的。

# BUG

  • 现在使用的是vuepress v2的beta版本，不知道是不是个例，写完内容保存后在dev环境下偶尔会出现报错，特别是在开启了google的分析之后，遇到这种情况只有重启dev服务器才能解决这个问题。

    Uncaught ReferenceError: GA_ID is not defined at clientAppEnhance.js:3:12

  • 使用deploy脚本更新，经常需要更新2次才成功，不知道是由于有缓存还是更新的比较慢产生了错觉。
