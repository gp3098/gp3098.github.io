# 学习ssh相关知识

remote_ssh_server是本地在~/.ssh/config内配置的主机名

# 本地端口转发

ssh -L 2111:www.google.com:80 remote_ssh_server -N
1
curl http://localhost:2111
1

访问本地的2111端口相当于访问了www.google.com进行了端口转发走的http

参考 https://wangdoc.com/ssh/port-forwarding.html

# 增加config配置

没有实践，懒得自己配置了，搬运一下吧

LocalForward client-IP:client-port server-IP:server-port

# 动态转发

# 方法1 直接通过命令

ssh  -D 22333 remote_ssh_server -N
1
curl -x socks5://localhost:22333 cip.cc
# -x 代表使用代理
1
2

在本地增加socks5代理后访问的所有网站或者其他流量都会转换成remote_ssh_server方式访问

# 方法2 增加config配置

在参考文档内没有详细的配置，重新参考了其他博主的文章才明白是怎么操作的

Host "remote_ssh_server"
    HostName xxx.xxx.xx.xx
    Port  22
    User ubuntu
    IdentityFile ~/.ssh/id_rsa
    DynamicForward localhost:22333
